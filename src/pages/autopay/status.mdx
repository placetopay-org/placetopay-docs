# Estados de un Autopago

| Estado   | Descripción                                                                                      |
|----------|--------------------------------------------------------------------------------------------------|
| APPROVED | Indica un estado de autopago aprobado.                                                           |
| REJECTED | Indica un estado de autopago rechazado. Puede estar relacionado con diferentes códigos de razón. |
| FAILED   | Indica un estado de autopago fallido. Puede estar relacionado con diferentes códigos de razón.   |

---

# Códigos de razón que puede tener un Autopago

El siguiente apartado describe los posibles códigos de respuesta que puede retornar el sistema de autopagos.
Estos códigos ayudan a comprender el estado deuna operación y permiten identificar con claridad posibles errores o condiciones especiales.


<Note type="warning">
Los códigos listados en esta sección no representan la totalidad de códigos que puede retornar el sistema.

El comercio integrador debe considerar que pueden recibirse códigos adicionales, incluyendo aquellos definidos en
el apartado general de [**Códigos de respuesta**](/autopay/codes), por lo que se recomienda implementar un manejo flexible
de respuestas y no restringir la integración exclusivamente a los códigos aquí descritos.
</Note>


| Reason Code | Descripción                                 |
|-------------|---------------------------------------------|
| 00          | Indica un proceso aprobado o satisfactorio. |
| ?C          | Indica un proceso cancelado.                |
| AP          | Indica un proceso de pagos culminado.       |

---

# Transición de estados de un autopago

En este apartado se indican los escenarios en los que un **autopago** puede cambiar de estado.

| Escenario                                                                                                                               | Estado   | Reason |
|-----------------------------------------------------------------------------------------------------------------------------------------|----------|--------|
| Creación de autopago.                                                                                                                   | APPROVED | 00     |
| Cancelación de un autopago.                                                                                                             | REJECTED | ?C     |
| Autopago completado.                                                                                                                    | REJECTED | AP     |
| Se alcanza el límite de pagos atrasados permitidos.                                                                                     | FAILED   | ?C     |
| Se alcanza el límite de reintentos por indisponibilidad del servicio externo en la consulta de balance durante el registro de un pago.  | FAILED   | ?C     |
| Se alcanza el límite de reintentos por fallo técnico, ya sea durante el registro o el procesamiento de la cuota de un autopago.         | FAILED   | ?C     |
