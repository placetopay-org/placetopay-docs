# Cómo funciona AutoPay

AutoPay simplifica la gestión de pagos recurrentes actuando como un orquestador central entre tu comercio, el usuario y el núcleo de procesamiento.

## Modelo de Integración

A diferencia de integraciones tradicionales, AutoPay ofrece un **Único Punto de Entrada (Gateway API)**. Tu comercio interactúa exclusivamente con el Gateway para definir las reglas de negocio (frecuencia, montos, fechas).

El Gateway actúa como un **Proxy** que se encarga de:
1.  **Orquestar:** Gestionar la sesión de interfaz (WebCheckout) para la captura segura de datos.
2.  **Proteger:** Tokenizar la información sensible bajo normativas PCI.
3.  **Persistir:** Guardar las reglas de recurrencia en el Core de AutoPay para su ejecución futura.

## Ciclo de Vida del AutoPago

1.  **Setup (API):** El comercio inicia una sesión enviando el objeto `recurring` al Gateway.
2.  **Captura (Front):** El usuario es redirigido a WebCheckout para ingresar sus datos de forma segura.
3.  **Ejecución (Batch):** El Core ejecuta los cobros automáticamente según la periodicidad definida. Si el cobro es variable, consultará tu API de Balance antes de procesar.
