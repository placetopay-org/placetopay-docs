# Cobro masivo

El servicio de **cobro masivo** permite procesar múltiples cobros recurrentes mediante archivos CSV cifrados. El comercio genera un archivo de entrada con las transacciones a ejecutar y recibe un archivo de respuesta con el estado final de cada cobro.

## ¿Cuándo usar este servicio?

Este mecanismo es recomendado cuando se necesita ejecutar una **gran cantidad de cobros de forma recurrente** y se desea automatizar el intercambio de información por archivos.

## Flujo general del proceso {{ id: 'charge-flow' }}

Antes de iniciar el procesamiento, el cliente debe compartir con Placetopay la información necesaria para ubicar y gestionar los archivos en su servicio SFTP:

- **Ruta de entrada:** directorio donde el cliente depositará los archivos que Placetopay debe descargar y procesar.
- **Patrón del nombre del archivo:** criterio utilizado para identificar los archivos a procesar. Puede definirse mediante un patrón simple, como `Visa*`, para tomar todos los archivos cuyo nombre inicie con `Visa`, o mediante una expresión regular compatible con la configuración acordada.
- **Ruta de salida:** directorio donde Placetopay depositará los archivos resultantes una vez finalizado el procesamiento.

<Note>
    El nombre del archivo no está limitado a una convención única. Placetopay procesará los archivos que coincidan con la ruta de entrada y el patrón de búsqueda previamente configurados.
</Note>

El proceso de integración se ejecuta de la siguiente forma:

1. Se establece la conexión entre servidores **SFTP**, mediante **VPN** o red pública.
2. Se generan y validan los datos de acceso para la comunicación **SFTP**.
3. Se define la periodicidad de procesamiento de archivos.
4. El cliente genera y comparte su llave pública **PGP/RSA** a Placetopay.
5. El cliente recibe la llave pública **PGP/RSA** de Placetopay.
6. El cliente genera un archivo CSV con la información necesaria para procesar los cobros.
7. El archivo se cifra con la llave pública de Placetopay usando **PGP**.
8. El cliente debe depositar el archivo en la ruta de entrada de su servicio SFTP, de acuerdo con el patrón del nombre compartido a Placetopay.
9. Placetopay descarga desde el SFTP del cliente los archivos que coincidan con la configuración definida.
10. Placetopay descifra el archivo.
11. Placetopay genera un archivo de resultados.
12. Placetopay cifra el archivo de resultados con PGP.
13. Placetopay deposita el archivo resultante en la ruta de salida definida por el cliente dentro de su servicio SFTP.
14. El cliente descifra la información.
15. El cliente actualiza su base de datos con los tokens y subtokens recibidos.

## Estructura del archivo de cobro {{ id: 'charge-input-file' }}

### Estructura del archivo CSV para cobrar

El archivo de entrada debe estar separado por comas, incluir encabezado y respetar los nombres de columna descritos a continuación.

<Row>
    <Col>
        <Properties>
            <Property name="siteId" type="int">
                Identificador del sitio proporcionado por Placetopay.
            </Property>
            <Property name="reference" type="string(32)">
                Referencia única del cobro.
            </Property>
            <Property name="instrumentType" type="string(1)">
                Tipo de instrumento a usar: `T` para Token o `S` para Subtoken.
            </Property>
            <Property name="instrument" type="string(64)">
                Token o subtoken utilizado para ejecutar el cobro.
            </Property>
            <Property name="currency" type="string(3)">
                Moneda conforme a ISO 421: `COP`, `USD` o `EUR`.
            </Property>
            <Property name="amount" type="decimal(14,2)">
                Monto definitivo a cobrar.
            </Property>
            <Property name="valueAddedTax" type="decimal(14,2)">
                Monto correspondiente al IVA.
            </Property>
            <Property name="exciseDuty" type="decimal(14,2)">
                Monto correspondiente al impuesto al consumo.
            </Property>
        </Properties>
    </Col>
    <Col sticky>

        ```txt {{ 'title': 'Ejemplo de archivo de entrada' }}
        siteId,reference,instrumentType,instrument,currency,amount,valueAddedTax,exciseDuty
        1001,KX00001,T,1860563980451111,COP,98185.71,15676.71,0
        ```

    </Col>
</Row>

<Note>
    Todos los campos son requeridos. Los valores decimales deben separarse con punto (`.`).
</Note>

## Estructura del archivo de respuesta {{ id: 'charge-response-file' }}

### Estructura del archivo CSV resultante

El archivo de respuesta contiene el resultado del procesamiento de cada cobro.

<Row>
    <Col>
        <Properties>
            <Property name="siteId" type="int">
                Identificador del sitio proporcionado por Placetopay.
            </Property>
            <Property name="reference" type="string(32)">
                Referencia del cobro.
            </Property>
            <Property name="status" type="string(20)">
                Estado de la transacción: `APPROVED`, `REJECTED` o `FAILED`.
            </Property>
            <Property name="iso" type="string(2)">
                Código del motivo de la respuesta en formato ISO 8583 Response Code Version 1987.
            </Property>
            <Property name="base24" type="string(3)">
                Código del motivo de respuesta en formato Base24.
            </Property>
            <Property name="currency" type="string(3)">
                Moneda en la cual se realizó el cobro: `COP`, `USD` o `EUR`.
            </Property>
            <Property name="amount" type="decimal(14,2)">
                Monto definitivo cobrado.
            </Property>
            <Property name="discount" type="decimal(10,2)">
                Descuento aplicado en la transacción.
            </Property>
            <Property name="internalReference" type="string(20)">
                Código interno de la transacción en Placetopay.
            </Property>
            <Property name="authorization" type="string(6)">
                Código de autorización de la transacción.
            </Property>
            <Property name="receipt" type="string(10)">
                Código de la transacción en la red.
            </Property>
        </Properties>
    </Col>
    <Col sticky>

        ```txt {{ 'title': 'Ejemplo de archivo de respuesta' }}
        siteId,reference,status,iso,base24,currency,amount,discount,internalReference,authorization,receipt
        1001,KX00001,APPROVED,00,000,COP,98185.71,1234.29,12345678901234567890,000000,0000000000
        ```

    </Col>
</Row>

<Note>
    Los valores decimales deben separarse con punto (`.`). Todos los campos son requeridos. El archivo contiene encabezado y los nombres del encabezado corresponden a los definidos en la propiedad `name`.
</Note>

## Reglas de formato {{ id: 'charge-format-rules' }}

| Regla | Descripción |
|---|---|
| Separador de columnas | Coma (CSV) |
| Encabezados | Obligatorios |
| Campos | Todos son requeridos |
| Decimales | Deben separarse con punto (`11244.13`) |
| Longitud de datos | Los tipos de datos proporcionados indican longitudes máximas; los archivos no son de longitud fija |


## Consideraciones operativas {{ id: 'charge-operational-notes' }}

- El servidor donde se realice el cargue y descarga de archivos debe ser accesible mediante **llaves RSA** para que el proceso pueda ejecutarse automáticamente.
- El tiempo de procesamiento, desde la descarga del archivo de entrada hasta la carga del archivo de respuesta, es variable y depende de la cantidad de transacciones, el estado de las redes, entre otros factores.
- Se puede crear un `cron job` en el servidor del comercio para verificar la disponibilidad del archivo de respuesta cada **30 minutos** a partir del cargue inicial.
