# Datos de petición a process transaction
En los datos adicionales se puede enviar información que se desee guardar con la transacción.
Las siguientes claves **NO** son permitidas: `_accountNumber`, `userAgent`, `fingerprint`, `sourcePlatform`,
`tokenizationID`, `trazabilyCode`, `transactionCycle`, `RequestId`, `PartnerAuthCode`, `onTest`, `threeDSEnrolled`,
`threeDSAuthenticated`.

Además algunos parámetros para el procesamiento pueden ser enviados en este campo:
### Indicador de iniciador de transacción
Si se necesita procesar con un valor particular de [indicador de iniciador de pago](./initiator-indicator), se puede enviar esta información
como un dato adicional.

```json
{
  //...
  "additional": {
    "initiatorIndicator": "CARDHOLDER_COF"
  }
}
```

# Datos en respuestas a process transaction

Existen datos adicionales que son recibidos en respuesta a la solicitud de procesamiento de una transacción. La recepción de estos datos depende de diversos factores, y viene en la respuesta como `additional`.

```json
{
    "status": {
        "status": "APPROVED",
        "reason": "00",
        "message": "Approved",
        "date": "2024-07-17T15:28:35-05:00"
    },
    // ...
    "additional": {
      // Datos adicionales
      "bin": "411076",
      "terminalNumber": "00990101",
      // ...
    }
}
```

## Crédito (Credit) {{ id: 'additional-data-credit' }}
Este campo especifica las opciones de crédito disponibles para financiar el monto total de un pago. La estructura del arreglo `credits` depende del proveedor y de la configuración del servicio.

Existen dos formas de informar créditos:

- **Estructura agrupada**: agrupa una misma condición de crédito en `installments` y usa `type`, `code` y `groupCode` como códigos del proveedor.
- **Estructura por cuota**: informa cada opción de financiación como un elemento independiente y usa `kind` como identificador funcional de la modalidad de crédito.

### Estructura agrupada:
- **type**: Identificador que representa el tipo de crédito.
- **groupCode**: Código que agrupa los créditos por categoría.
- **installments**: Cantidad de cuotas disponibles para el crédito.
- **code**: Código único asociado al crédito específico.
- **description**: Descripción detallada de las condiciones de financiamiento.

### PAYSTUDIO flavour TS con estructura por cuota  {{ id: 'additional-data-credit-providers-paystudio-ts' }}
Esta estructura está disponible por ahora solo para servicios PAYSTUDIO con `flavour` igual a `TS`. Los demás proveedores y otros valores de `flavour` en PAYSTUDIO continúan usando la estructura agrupada descrita anteriormente.

En PAYSTUDIO flavour `TS`, `credits` se entrega sin `type`, `code`, `groupCode` ni `installments`. Cada elemento representa una sola opción de financiación.

#### **Estructura:**
- **installment**: Número de cuotas de la opción.
- **description**: Texto visible para el pagador.
- **kind**: Clasificación de la modalidad de crédito. Este valor reemplaza los códigos de red al momento de seleccionar y procesar la cuota.

```json
[
  {
    "installment": 1,
    "description": "Sin cuotas",
    "kind": "ISSUER_NOT_FEE"
  },
  {
    "installment": 2,
    "description": "Cuotas sin interés",
    "kind": "MERCHANT_WITHOUT_INTEREST"
  },
  {
    "installment": 3,
    "description": "Cuotas sin interés",
    "kind": "ISSUER_WITHOUT_INTEREST"
  },
  {
    "installment": 6,
    "description": "Cuotas",
    "kind": "ISSUER_UNKNOWN_01"
  }
]
```

En esta estructura, el integrador debe enviar el `kind` de la opción seleccionada dentro de `instrument.credit` junto con el número de cuota:

```json
{
  "instrument": {
    "credit": {
      "installment": 6,
      "kind": "ISSUER_UNKNOWN_01"
    }
  }
}
```

Si se envía `kind` junto con `type` o `code`, `kind` tiene prioridad para servicios PAYSTUDIO flavour `TS` con esta estructura. Si no se envía `credit`, se usará el crédito por defecto de una cuota.

#### **Valores de `kind`**
El valor de `kind` se construye con el formato `ORIGEN_MODALIDAD`:

| Origen | Significado |
|--------|-------------|
| `MERCHANT` | Crédito del comercio |
| `ISSUER` | Crédito del emisor |
| `ACQUIRER` | Crédito del adquirente |
| `AVERAGE` | Crédito promedio |

| Modalidad | Significado |
|-----------|-------------|
| `NOT_FEE` | Pago sin cuotas o crédito corriente |
| `WITHOUT_INTEREST` | Cuotas sin interés |
| `UNKNOWN_XX` | Cuotas cuya condición de interés debe resolverse con el subtipo `XX` del proveedor |

#### **Ejemplos de equivalencia:**

| salePlan | subType | installment | kind |
|----------|---------|-------------|------|
| `01` | `00` | `1` | `MERCHANT_NOT_FEE` |
| `01` | `00` | `2` o más | `MERCHANT_WITHOUT_INTEREST` |
| `02` | `02` | `1` | `ISSUER_NOT_FEE` |
| `02` | `04` | `2` o más | `ISSUER_WITHOUT_INTEREST` |
| `02` | `00` | `2` o más | `ISSUER_UNKNOWN_00` |
| `02` | `01` | `2` o más | `ISSUER_UNKNOWN_01` |
| `02` | `03` | `2` o más | `ISSUER_UNKNOWN_03` |
| `03` | `00` | `2` o más | `ACQUIRER_UNKNOWN_00` |
| `04` | `00` | `2` o más | `AVERAGE_UNKNOWN_00` |

Para créditos `UNKNOWN_XX` en PAYSTUDIO flavour `TS`, usa el servicio de intereses antes de procesar cuando sea necesario conocer el valor final de la cuota. Si el servicio no puede determinar la tasa aplicable, la solicitud puede ser rechazada con razón `NR`.
