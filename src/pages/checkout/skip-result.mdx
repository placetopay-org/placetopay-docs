# Omitir resultado
La API de Web Checkout permite omitir o saltar la vista de resumen de una sesión con un proceso en [estado finalizado](/checkout/how-checkout-works#checkout-session-states). Esta funcionalidad es útil si la integración cuenta con su propia vista personalizada del resumen de la sesión y no se necesita que Web Checkout la muestre.

Para habilitar esta funcionalidad, se debe incluir la propiedad `skipResult` con el valor `true` en la solicitud de [creación de una sesión](/checkout/create-session). Si no se envía esta propiedad o se establece con un valor diferente, se tomará por defecto como `false`.

```json
{
    "locale": "es_CO",
    "auth": {...},
    "payment": {...},
    "skipResult": true,
    "expiration": "2021-12-30T00:00:00-05:00",
    "returnUrl": "https://dnetix.co/p2p/client",
    "ipAddress": "127.0.0.1",
    "userAgent": "PlacetoPay Sandbox"
}
```

## ¿Cómo funciona? {{ id: 'behavior-api' }}
Cuando el proceso de una sesión es finalizado, el usuario es redirigido a la URL especificada en la propiedad `returnUrl` incluida en la solicitud original de creación de la sesión sin necesidad de interacción por parte del usuario.

## ¿Cómo funciona usando Lightbox SDK? {{ id: 'behavior-lihtbox' }}
Para [integraciones con Lightbox](/checkout/lightbox#advanced-integration)se despacha el evento `close` y  la ventana del Lightbox se cierra automaticamente sin necesidad de interacción por parte del usuario.
