# Límite de intentos por sesión   
WebCheckout cuenta con una funcionalidad para limitar la cantidad de **intentos fallidos** (transacciones o suscripciones con estado **rechazado**) durante el procesamiento de una sesión. Esta medida busca proteger al usuario final y evitar múltiples intentos inválidos o maliciosos durante una misma sesión de pago.
Este límite **no se basa en la cantidad total de transacciones**, sino únicamente en aquellas que hayan sido rechazadas por el sistema o el medio de pago.
Cuando la cantidad de intentos fallidos alcanza el límite configurado por defecto 10, la sesión se marca automáticamente con estado [REJECTED](/checkout/how-checkout-works#checkout-session-states) y se impide continuar con nuevos procesos dentro de esa misma sesión.

## ¿Cuándo se aplica?
Esta funcionalidad se activa automáticamente durante el procesamiento de la sesión, ya sea mediante integración directa por API o a través de Lightbox. El sistema lleva un conteo de los intentos rechazados y, al superar el límite configurado, la sesión se da por finalizada y no permite realizar más operaciones.

## ¿Qué incluye como intento rechazado?
Se consideran intentos fallidos únicamente los procesos de:
- Transacciones con estado rechazado
- Suscripciones con estado rechazado

No se contabilizan procesos con estados aprobados, pendientes, o fallidos por error de red.
