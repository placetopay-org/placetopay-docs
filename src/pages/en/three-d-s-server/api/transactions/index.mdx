# Transaction Show


**This is the service for obtaining information about a transaction that used the cardholder authentication flow with the 3DS Server.**

The following steps will provide you with a guide for its usage:

  **1.** Create an HTTP GET request on your server with the endpoint /api/v2x/transactions/transactionID

  **2.** Append the transaction identifier as a parameter at the end of the URL, which corresponds to the transactionID field obtained in the response from the service with the endpoint api/threeds/v2x/sessions

  **3.** In the request header, include the Bearer token, which you can create by navigating to the merchant section and then to the Tokens section. This will allow you to register as an authorized user and is mandatory.

  **4.** The header must also contain the Content-Type with the value "application/json".

  **5.** This request does not require data to be passed in the request body.

  **6.** Send the request and obtain a response.


***The description of the response fields, data type, length, and other specifications can be viewed in the responses section, in the data schema.***

See more details in [Values returned in the API response](/en/three-d-s-server/api/transactions/values)
