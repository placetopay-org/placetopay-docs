# Test Cards

The following credit card numbers allow you to make requests to the service and obtain the specific behaviors described for each one. The possible states are:

* **Y**: Frictionless authenticated
* **A**: Authentication Attempt (see note)
* **N**: Not authenticated
* **R**: Rejected
* **C**: Challenge
* **D**: Decoupled Authentication
* **I**: Information Only (see note)

<Note>
The states **A**: _Authentication Attempt_ and **I**: _Information Only_ are internal application states used for certification testing; they do not have behavior outside of this context.

To see the Authorization behaviors for the cards, go to: [Gateway API - Test Card Numbers](/en/gateway/testing-card)
</Note>
