# Webhooks

A webhook is a notification sent to a URL via an HTTP request. In Placetopay, webhooks allow external systems to receive real-time information about the status of a transaction. {{ className: 'lead' }}

Currently, several types of webhooks are supported, each with a specific purpose and different configuration. The available webhooks are:

- Webhook defined in the session
- Webhook for ACH returns

## Webhook defined in the session

When creating a payment session, it is possible to configure a notification URL. Whenever an update occurs in the session (session approved, transaction reversed, etc.), a webhook will automatically be sent to this URL.

- **Purpose**: Notify the merchant when the status of a session changes.
- **Configuration**: Defined when creating the session by including the `notificationURL` parameter in [Gateway - Process request](http://localhost:3000/gateway/api/reference/transaction#process-request).
- **Event**: When a session is updated (e.g., pending to approved, approved to reversed, etc.).
- **Information sent**: Basic session information such as status, reason, internalReference, reference, among others.

### Signature validation

To ensure the authenticity of the notification, the session webhook includes a signature in the `X-Signature` header. You can validate this signature using an SHA-1 hash generated with the values of the session ID, session status, and the transaction key (`tranKey`) of your site.

```javascript {{ title: 'Validation in JavaScript' }}
const crypto = require('crypto');

const sessionId = notificationBody.session.id;
const sessionStatus = notificationBody.session.status;
const tranKey = 'YOUR_SITE_TRANKEY';

const signatureString = `${sessionId}${sessionStatus}${tranKey}`;

// Generate the signature locally
const generatedSignature = crypto
  .createHash('sha1')
  .update(signatureString)
  .digest('hex');

// Signature received in the notification
const receivedSignature = headers['X-Signature'];

// Validate the signature
if (generatedSignature === receivedSignature) {
  console.log("The notification is authentic.");
} else {
  console.log("Invalid signature: the notification is not authentic.");
}
```

## Webhook for ACH returns

The webhook for ACH returns allows merchants to receive real-time notifications about returns on their site.

- **Purpose**: Notify the merchant when an ACH return occurs.
- **Configuration**: Configured in the Placetopay administrative panel through the operations team, providing the desired URL.
- **Event**: HTTP notification sent when an ACH return is generated.
- **Information sent**: All direct session-related information.

```plaintext {{ title: 'Headers' }}
X-Signature: [signature for authenticity validation]
Content-Type: application/json
Accept: application/json
```

### Signature validation

To ensure the authenticity of the notification, each webhook from Placetopay includes a signature in the `X-Signature` header. You can validate this signature using an HMAC SHA-256 hash generated with the content of the notification body (`body`) and the transaction key (`tranKey`) of your site.

```javascript {{ title: 'Validation in JavaScript' }}
const crypto = require('crypto');

// Body of the request and received signature
const body = JSON.stringify(notificationBody);
const receivedSignature = headers['X-Signature'];

const tranKey = 'YOUR_SITE_TRANKEY';

// Generate the signature locally
const generatedSignature = crypto
  .createHmac('sha256', tranKey)
  .update(body)
  .digest('hex');

// Validate the signature
if (generatedSignature === receivedSignature) {
  console.log("The notification is authentic.");
} else {
  console.log("Invalid signature: the notification is not authentic.");
}
```

## FAQs

<details>
<summary>How do I configure a webhook for my business?</summary>
<p>For the session-defined webhook, include the <code>notificationURL</code> parameter when creating a payment session. For the ACH returns webhook, contact the operations team and provide the desired URL.</p>
</details>

<details>
<summary>What information is sent by the webhooks?</summary>
<p>The session-defined webhook includes the session status, reference, and internalReference. The ACH returns webhook includes details such as the return reason, amount, and associated references.</p>
</details>

<details>
<summary>How can I validate the authenticity of webhooks?</summary>
<p>Validate the signature provided in the <code>X-Signature</code> header using HMAC SHA-256 or SHA-1, depending on the webhook type.</p>
</details>
