# Massive Recurring

The **massive recurring** service allows multiple recurring charges to be processed through encrypted CSV files. The merchant generates an input file containing the transactions to be executed and receives a response file with the final status of each charge.

## When should this service be used?

This mechanism is recommended when a **large number of recurring charges** must be executed and the file-based information exchange needs to be automated.

## General process flow {{ id: 'charge-flow' }}

Before processing begins, the customer must share with Placetopay the information required to locate and manage files within its SFTP service:

- **Input path:** directory where the customer will place the files that Placetopay must download and process.
- **File name pattern:** criterion used to identify the files to be processed. It may be defined through a simple pattern, such as `Visa*`, to include all files whose names begin with `Visa`, or through a regular expression compatible with the agreed configuration.
- **Output path:** directory where Placetopay will place the resulting files once processing is complete.

<Note>
    The file name is not limited to a single naming convention. Placetopay will process the files that match the previously configured input path and file search pattern.
</Note>

The integration process is carried out as follows:

1. The connection between **SFTP** servers is established through **VPN** or a public network.
2. Access credentials for **SFTP** communication are generated and validated.
3. The file processing frequency is defined.
4. The customer generates and shares its **PGP/RSA** public key with Placetopay.
5. The customer receives Placetopay's **PGP/RSA** public key.
6. The customer generates a CSV file with the information required to process the charges.
7. The file is encrypted with Placetopay's public key using **PGP**.
8. The customer must place the file in the input path of its SFTP service, according to the file name pattern shared with Placetopay.
9. Placetopay downloads from the customer's SFTP the files that match the defined configuration.
10. Placetopay decrypts the file.
11. Placetopay generates a results file.
12. Placetopay encrypts the results file with PGP.
13. Placetopay places the resulting file in the output path defined by the customer within its SFTP service.
14. The customer decrypts the information.
15. The customer updates its database with the received tokens and subtokens.

## Charge file structure {{ id: 'charge-input-file' }}

### CSV file structure for charges

The input file must be comma-separated, include headers, and preserve the column names described below.

<Row>
    <Col>
        <Properties>
            <Property name="siteId" type="int">
                Site identifier provided by Placetopay.
            </Property>
            <Property name="reference" type="string(32)">
                Unique charge reference.
            </Property>
            <Property name="instrumentType" type="string(1)">
                Instrument type to be used: `T` for Token or `S` for Subtoken.
            </Property>
            <Property name="instrument" type="string(64)">
                Token or subtoken used to execute the charge.
            </Property>
            <Property name="currency" type="string(3)">
                Currency according to ISO 421: `COP`, `USD`, or `EUR`.
            </Property>
            <Property name="amount" type="decimal(14,2)">
                Final amount to be charged.
            </Property>
            <Property name="valueAddedTax" type="decimal(14,2)">
                Amount corresponding to VAT.
            </Property>
            <Property name="exciseDuty" type="decimal(14,2)">
                Amount corresponding to excise tax.
            </Property>
        </Properties>
    </Col>
    <Col sticky>

        ```txt {{ 'title': 'Input file example' }}
        siteId,reference,instrumentType,instrument,currency,amount,valueAddedTax,exciseDuty
        1001,KX00001,T,1860563980451111,COP,98185.71,15676.71,0
        ```

    </Col>
</Row>

<Note>
    All fields are required. Decimal values must use a period (`.`).
</Note>

## Response file structure {{ id: 'charge-response-file' }}

### Resulting CSV file structure

The response file contains the processing result for each charge.

<Row>
    <Col>
        <Properties>
            <Property name="siteId" type="int">
                Site identifier provided by Placetopay.
            </Property>
            <Property name="reference" type="string(32)">
                Charge reference.
            </Property>
            <Property name="status" type="string(20)">
                Transaction status: `APPROVED`, `REJECTED`, or `FAILED`.
            </Property>
            <Property name="iso" type="string(2)">
                Response reason code in ISO 8583 Response Code Version 1987 format.
            </Property>
            <Property name="base24" type="string(3)">
                Response reason code in Base24 format.
            </Property>
            <Property name="currency" type="string(3)">
                Currency in which the charge was processed: `COP`, `USD`, or `EUR`.
            </Property>
            <Property name="amount" type="decimal(14,2)">
                Final amount charged.
            </Property>
            <Property name="discount" type="decimal(10,2)">
                Discount applied to the transaction.
            </Property>
            <Property name="internalReference" type="string(20)">
                Internal transaction code in Placetopay.
            </Property>
            <Property name="authorization" type="string(6)">
                Transaction authorization code.
            </Property>
            <Property name="receipt" type="string(10)">
                Transaction code in the network.
            </Property>
        </Properties>
    </Col>
    <Col sticky>

        ```txt {{ 'title': 'Response file example' }}
        siteId,reference,status,iso,base24,currency,amount,discount,internalReference,authorization,receipt
        1001,KX00001,APPROVED,00,000,COP,98185.71,1234.29,12345678901234567890,000000,0000000000
        ```

    </Col>
</Row>

<Note>
    Decimal values must use a period (`.`). All fields are required. The file includes headers, and the header names correspond to those defined in the `name` property.
</Note>

## Formatting rules {{ id: 'charge-format-rules' }}

| Rule | Description |
|---|---|
| Column separator | Comma (CSV) |
| Headers | Required |
| Fields | All fields are required |
| Decimals | Must use a period as the separator (`11244.13`) |
| Data length | The provided data types indicate maximum lengths; files are not fixed-width |

## Operational considerations {{ id: 'charge-operational-notes' }}

- The server where files are uploaded and downloaded must be accessible through **RSA keys** so the process can run automatically.
- Processing time, from downloading the input file to uploading the response file, is variable and depends on the number of transactions, network conditions, and other factors.
- A `cron job` can be created on the merchant's server to check for the availability of the response file every **30 minutes** after the initial upload.
