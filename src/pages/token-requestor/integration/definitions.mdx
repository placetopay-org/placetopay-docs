# Definiciones clave

## Consumidor - _**Customer**_

Es la entidad principal que representa a una pasarela de pagos o algún cliente externo que
posee uno o varios comercios registrados en el sistema. Este consumidor puede ser una
empresa o individuo responsable de gestionar las credenciales de acceso y los comercios
suscritos a Token Requestor.


## Comercio de la Marca -  _**Merchant Brand**_

Es la representación del comercio ante la marca **VISA** o **MASTERCARD**. Un comercio
puede tener múltiples entradas en Merchant Brand, una por cada marca que acepte. Esta
entidad nace del proceso de onboarding con la marca, donde el comercio es registrado ante
las marcas para la generación de tokens.
Si el comercio está registrado en varios consumidores, solo existirá un registro de comercio
ante las marcas.


## Comercio -  _**Merchant**_

Representa a un comercio específico vinculado a un consumidor. Es la unidad a la cual se
le generarán los cobros de la generación y gestión de tokens. Para que un comercio pueda
gestionar tokens deberá estar asociado a un comercio de la marca **Merchant Brand** activo.

## Credenciales
Representan los datos de autenticación y autorización vinculados a un consumidor. Estas
credenciales permiten el acceso seguro a la API y están diseñadas para gestionar tokens de
marca únicamente para los comercios asociados a dicho consumidor.
Además, el consumidor tiene la capacidad de generar múltiples credenciales propias,
aplicando filtros personalizados **include o exclude** que le permiten definir con mayor
precisión qué comercios pueden gestionar tokens de marca con esas credenciales. Esto le
brinda un control más granular sobre la operación y seguridad de sus propios comercios.
