# Sitios

La API de Sitios está pensada para facilitar la gestión de los sitios que pertenecen a un comercio-consumidor dentro del entorno administrativo. Estos sitios funcionan como puntos de integración, permitiendo que cada comercio pueda conectarse con los servicios de tokenización y autorización de forma independiente y controlada.

Desde esta API es posible registrar nuevos sitios, consultarlos de manera individual o en listados paginados, actualizarlos cuando haya cambios en su configuración o eliminarlos si ya no son necesarios. Cada sitio siempre está vinculado a un merchant (Comercio) y a un consumer (Consumidor), y puede configurarse con parámetros como su nombre, URLs de notificación, credenciales de seguridad y fechas de habilitación.

La idea es brindar a los administradores una forma clara y segura de mantener organizados estos puntos de conexión. En los apartados siguientes se explican cada una de las operaciones disponibles, con los campos necesarios, las reglas de validación que aplican y ejemplos de cómo luce la respuesta de cada solicitud.

<Note> Todas las operaciones requieren autenticación por medio de credenciales válidas; se puede consultar más detalles en la sección de autenticación de esta documentación. </Note>

## Creación de sitios {{ id: 'token-requestor-api-admin-sites-create', tag: 'POST', label: '/api/admin/sites/create' }}


Esta operación permite registrar un nuevo sitio dentro del entorno de administración del sistema. El servicio está disponible únicamente para usuarios autenticados con permisos asociados y requiere una credencial válida para asociar el sitio con un merchant autorizado. Esta operación se utiliza principalmente en entornos administrativos, tanto de prueba como de producción, dependiendo de los valores suministrados.

Una vez creado, el sitio queda registrado en el sistema con su respectiva configuración, habilitación y, si aplica, parámetros de webhook para recepción de notificaciones. También se puede registrar en modo prueba (onTest), lo cual influye en su comportamiento operativo.

## Consulta de sitios {{ id: 'token-requestor-api-admin-sites-show', tag: 'POST', label: '/api/admin/sites/show' }}

Esta operación permite consultar la información detallada de un sitio previamente registrado en el sistema administrativo. Está disponible exclusivamente para usuarios autenticados que tengan permisos de acceso y visibilidad sobre el recurso.

Está diseñada para actualizar únicamente los campos enviados en la solicitud, sin alterar los demás valores del sitio. ## Listado de sitios {{ id: 'token-requestor-api-admin-sites-index', tag: 'POST', label: '/api/admin/sites/index' }}

Esta operación permite obtener un listado de los sitios registrados en el sistema administrativo retornando un conjunto de resultados paginados, con la posibilidad de aplicar múltiples filtros para acotar la búsqueda. Solo puede ser utilizada por credenciales autenticadas y autorizadas, de acuerdo con los permisos de acceso definidos por el sistema.

El resultado de esta operación incluye información básica de cada sitio, como su identificador, datos del consumidor y merchant relacionados, nombre, URLs de webhook, fechas clave y su estado de prueba o producción.

El sistema soporta además mecanismos de exclusión por ACL, por lo cual ciertos sitios pueden no aparecer en el listado dependiendo de las reglas de acceso configuradas para la credencial autenticada.
